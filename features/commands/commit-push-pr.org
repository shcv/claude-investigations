#+TITLE: /commit-push-pr Command
#+DATE: 2026-02-26

* Overview

The =/commit-push-pr= command automates the full git workflow: create a branch,
stage changes, commit, push, and open or update a pull request — in one step.
The resulting PR includes attribution statistics showing how much of the work
was AI-assisted.

* Usage

Type =/commit-push-pr= in the Claude Code prompt. Claude Code will analyze the
current git state and execute the full branch/commit/push/PR workflow.

* What It Does

** Prompt Generation

=/commit-push-pr= works by injecting a detailed prompt into a Claude turn rather
than running git commands directly. The prompt (=w3q=, lines 456518–456590)
includes:

- Current =git status= and =git diff= output
- The current base branch (from =_N()=)
- A git safety protocol (never force-push main, never skip hooks, etc.)
- Explicit instructions to: create branch → stage → commit → push → create/update PR

** Branch Naming

Branch names follow the pattern =SAFEUSER/feature-name= where =SAFEUSER= is
derived from the environment (=whoami= or equivalent). If that's unavailable,
it falls back to =whoami=.

** Attribution Statistics

The PR body includes an attribution block generated by =uKq= (lines
455785–455820):

#+begin_src text
🤖 Generated with Claude Code (42% 5-shotted by claude-opus-4-6, 2 memories recalled)
#+end_src

Components:
- *=claudePercent=*: percentage of the diff attributable to Claude (from
  =PoY()=, line 455808)
- *=promptCount=*: number of user prompts in this session ("N-shotted")
- *=memoryAccessCount=*: number of memory files recalled during the session
- *Model name*: the Claude model used

** PR Body Format

The generated PR uses a structured markdown body (lines 456565–456583):

#+begin_src markdown
## Summary
- <1-3 bullets>

## Test plan
- [ ] <checklist items>

## Changelog
<optional, if relevant changes exist>

🤖 Generated with Claude Code (42% 5-shotted by claude-opus-4-6, 2 memories recalled)
#+end_src

If a =commit.attribution= setting exists, an additional =Co-Authored-By= block
is appended (lines 456549–456559).

** Slack Integration

After PR creation, the command searches for available Slack tools via
=ToolSearch= and, if found, offers to post the PR link to Slack (line 456588).
Claude asks the user before sending.

* Allowed Tools

The command only grants access to a limited toolset (lines 456598–456610):
- Git operations: =git checkout=, =git add=, =git push=, =git commit=
- GitHub CLI: =gh pr= commands
- =ToolSearch= (for Slack discovery)
- Slack tools (if found)

This prevents the PR-creation turn from using unrelated tools.
